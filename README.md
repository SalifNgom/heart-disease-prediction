# heart-disease-prediction
Projet de machine learning pour prédire le risque de maladie cardiaque à partir de données cliniques
## 1. Présentation du projet
Ce projet a pour objectif de développer un modèle de machine learning capable de prédire la
présence d’une maladie cardiaque à partir de données cliniques.
Le jeu de données utilisé est le Heart Disease Prediction Dataset, contenant plusieurs variables
médicales telles que l’âge, le cholestérol, la pression artérielle et d’autres indicateurs cliniques.
L’objectif est d’identifier les patients à risque afin d’illustrer l’intérêt des méthodes de data science
dans l’aide à la décision médicale.
## 2. Objectifs- 
Réaliser une analyse exploratoire des données- Préparer les données pour l’apprentissage automatique- Entraîner un modèle de classification- Évaluer les performances du modèle- Interpréter les résultats dans un contexte médical
## 3. Méthodologie
Le projet suit les étapes classiques d’un pipeline de machine learning :- Exploration et visualisation des données- Prétraitement et standardisation des variables- Séparation en ensembles d’entraînement et de test- Entraînement du modèle de classification- Évaluation via la matrice de confusion et les métriques de performance
Les performances du modèle ont été évaluées à l’aide des indicateurs suivants :- Accuracy- Precision- Recall
## 4. Résultats
Le modèle a obtenu les performances suivantes :- Accuracy : 83 %
- Precision : 83 %- Recall : 80 %
Ces résultats montrent que le modèle détecte correctement la majorité des patients à risque.
Cependant, certains faux négatifs subsistent, ce qui souligne l’importance d’améliorer la sensibilité
du modèle dans un contexte médical.
## 5. Conclusion
Ce projet démontre l’intérêt du machine learning pour la prédiction du risque cardiaque.
Il met également en évidence l’importance d’interpréter les résultats au-delà de l’accuracy globale,
en accordant une attention particulière au rappel (recall), essentiel dans les applications médicales.
Des améliorations futures pourraient inclure :- l’optimisation des hyperparamètres- l’utilisation de modèles plus robustes comme Random Forest ou XGBoost- l’ajustement du seuil de classification pour réduire les faux négatifs
